# 2026-03-22 Tassadar Starter-Plugin User Authoring Wave

`TAS-227` closes the first published starter-plugin user-authoring wave above
the shared starter-plugin runtime.

## Landed Surfaces

- the central runtime-owned starter-plugin registry in
  `docs/TASSADAR_STARTER_PLUGIN_RUNTIME.md`
- the bounded authoring contract in
  `docs/TASSADAR_STARTER_PLUGIN_AUTHORING.md`
- the shared bridge in `docs/TASSADAR_STARTER_PLUGIN_TOOL_BRIDGE.md`
- the starter catalog in `docs/TASSADAR_STARTER_PLUGIN_CATALOG.md`
- the deterministic workflow controller in
  `docs/TASSADAR_STARTER_PLUGIN_WORKFLOW_CONTROLLER.md`
- the canonical weighted-controller handoff via the updated weighted trace
  artifacts

## What Is Green

- one dependency-ordered user-authoring path instead of repeated manual
  per-surface edits
- one real user-added capability-free starter plugin, `plugin.text.stats`,
  propagated from runtime truth to the canonical weighted controller lane
- one published template and scaffold helper for the low-risk authoring class

## What Is Still Refused

- public plugin marketplace posture
- arbitrary external binary loading
- automatic widening of every new plugin into every controller surface
- networked authoring as the default path

## Next Frontier

The next frontiers remain later publication, trust-tier, and broader controller
or platform-widening work, not more hidden starter-plugin metadata duplication.
