[[SECTION:wasm_core_spec_release_2:2.5.1|2.5.1 Validation Contexts]]
validation_context defines the assumptions that instruction_validation may rely on and it must stay explicit in a normative explanation.

[[SECTION:wasm_core_spec_release_2:2.5.4|2.5.4 Malformed Module Conditions]]
malformed_module_condition names the boundary where a module is rejected before execution and the text forbids treating that boundary as scheduler commentary.
